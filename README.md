# RecipeStudio

RecipeStudio — desktop‑приложение на **Avalonia + .NET 8** для ведения технологических рецептов: хранение в SQLite, редактирование точек, автоматический пересчёт координат/импульсов, визуальная проверка траектории и импорт/экспорт в Excel.

---

## 1) Что это за приложение и зачем оно нужно

Приложение решает практическую задачу: взять «рецепт» (набор точек и параметров), пересчитать машинные/роботные координаты по формулам, сохранить результат и быстро проверить траекторию визуально.

Типовой сценарий:
1. Открываете рецепт или создаёте новый.
2. Редактируете точки (`R`, `Z`, углы, скорость, зона `Safe`, `place` и т.д.).
3. Нажимаете **«Пересчитать»**.
4. Получаете рассчитанные поля (`Xr0`, `dX`, `Xpuls`, `TopHz`, `ClampPuls` и др.).
5. Проверяете профиль в **PROFILE VIEWER (R‑Z)** и при необходимости подправляете точки.
6. Сохраняете в БД или экспортируете в Excel.

## 2) Быстрый старт

### Требования
- .NET SDK 8.0+
- ОС: Windows/Linux/macOS (Avalonia cross-platform)

### Запуск
```bash
cd src/RecipeStudio.Desktop
dotnet restore
dotnet run
```

### Публикация (single-file)
```bash
cd src/RecipeStudio.Desktop
dotnet publish -c Release
```

По умолчанию проект настроен на self-contained single-file публикацию для Win-x64.

## 3) Основные разделы интерфейса

## Dashboard
- список рецептов;
- создание/открытие/удаление.

## Редактор
- таблица точек рецепта;
- кнопки: добавить/дублировать/удалить/переместить/пересчитать;

- системные настройки хранения и логирования.

## 4) Как устроен пересчёт (кратко)

Пересчёт вызывается кнопкой **«Пересчитать»** и выполняется для всех точек:

1. Нормализация/служебные шаги
- синхронизация полей рецепта;
- проверка нумерации точек;
- расчёт вспомогательных значений (время, скорость сопла, рекомендованный расход).

2. CALC/SAVE поля
- преобразование углов в радианы;
- вычисление целевой точки `(Xp, Zp)` с учётом `place`;
- вычисление роботных координат `(Xr0, Yx0, Zr0)`;
- вычисление смещений `(dX, dY, dZ)`;
- импульсы по осям и частоты вращения;
- импульсы зажима по выбранному диаметру (форма/контейнер).

То есть после редактирования исходных полей вы всегда получаете согласованный набор производных полей.

## 5) PROFILE VIEWER (R‑Z): что показывается на графике

Визуализация ориентирована на «шаг 2» из Excel-схемы:

- целевые точки `(Xp, Zp)` для **Safe=0** — зелёный контур;
- целевые точки `(Xp, Zp)` для **Safe=1** — чёрный контур;
- контуры разделяются также по `place=0` и `place=1`;
- точки/траектория робота `(Xr, Zr)` — оранжевые;
- парные линии между `(Xp, Zp)` и `(Xr, Zr)` для `Safe=0` (контроль «сопло ↔ целевая точка»);
- маркер текущей позиции инструмента при проигрывании.
- интерактивный масштаб: кнопки `+`, `-`, `Fit` и колесо мыши в окне графика.
- легенду и вспомогательные связи можно отключать кнопками внизу графика, чтобы оставить «чистый» контур.

### Улучшение читаемости графика
Сделано специально для лучшей разборчивости:
- увеличен размер панели визуализации по умолчанию;
- убрана жёсткая маленькая высота графика — теперь он растягивается по панели;
- адаптивный внутренний отступ графика (на маленьком окне не «съедает» полезную область);
- легенда на полупрозрачной подложке.

## 6) Хранение данных

- Основное хранилище: SQLite (`recipes.sqlite`).
- Импорт/экспорт: Excel (`.xlsx`), также поддержка TSV/CSV совместимых потоков.
- Поля точки хранят как исходные, так и рассчитанные значения — это упрощает обмен с внешними файлами и контроль расчётов.

## 8) Практические рекомендации

- После любых изменений в таблице точек сначала нажимайте **«Пересчитать»**, потом проверяйте график.
- Если контур «не похож» на ожидаемый:
  1. проверьте `Safe` и `place` у точек;
  2. проверьте константы в настройках (`HZone`, `Lz`, импульсы);
  3. сравните несколько контрольных точек с Excel (R/Z, Xr/Zr, dX/dZ).
- Для визуального контроля удобнее увеличить панель визуализации (теперь это работает лучше и по умолчанию она больше).
- Если на графике кажется, что появилась «сетка из точек/линий», обычно это слой вспомогательных связей `Xp/Zp ↔ Xr/Zr`; его можно отключить кнопкой `Связи`.

## 9) Частые проблемы

- **`dotnet: command not found`** — не установлен .NET SDK.
- **Пустой график** — нет точек или они скрыты/некорректны.
- **Точка двигается «зеркально»** — проверьте `place`, для верхней половины используется зеркалирование.
